① Header carries the class and industry line in monospace at 8px. CEO name uses Playfair 16px bold. Archetype is italic muted text — a one-line identity summary.
② Four stat cells: Income/yr, Starting Stress (with pips), Death Roll threshold, T3 Access. Stats share the warm #faf8f2 background with the header to form a unified top block.
③ T3 Access section is only present when the CEO has a non-default rule. Most CEOs require owning a T2 first — only state the exception.
④ Ability tags encode type (PASSIVE / ONCE/YR / TRIGGERED / RISK) and stress cost. Stress cost is a critical decision point — it gets a red badge.
⑤ Labor Phase only shown when the CEO has a non-standard rule. Describes the d6 trigger range, bonus amount, and held card sale value override.
⑥ Starter asset renders as an inset mini-card — same layout language as market assets. Discard rule and transfer effect sit at the bottom of the inset.

A Header block — warm #faf8f2 background spans name + stats. Class and industry sit above the name as a monospace label. Archetype is always italic.
B Stat cells — four fixed slots: Income (green), Stress pips + number (red), Death Roll threshold (red), T3 Access label (amber if special, grey if default).
C T3 Access section — dashed amber left-border marks it as conditional. Omit entirely when the CEO uses the default T2-first requirement.
D Ability tags — PASSIVE (blue), ONCE/YR or TRIGGERED (amber), RISK (red). Stress cost always gets a red badge when non-zero.
E Starter asset inset — renders as a T0 mini-card. Always shows value, income, loan capacity. Discard rule at bottom. Family Trust variant is permanent (no discard note).
